Project Brief

Project Name: Pharmiq

Project Type:
Desktop-based Pharmaceutical Distribution Management System

Project Overview:
Pharmiq is a multi-user pharmaceutical distribution software designed for distributors who supply medicines to medical stores, clinics, and pharmacies. The system manages billing, inventory, customers, and compliance-related details like GST and drug licenses.

Core Concept:
- One distributor (user account) operates the system.
- Each distributor can have multiple login users (roles like Admin, Biller, Accountant).
- Each distributor manages their own customers (pharmacies, clinics — NOT walk-in customers).
- The system must support multi-tenant architecture where each distributor’s data is isolated.

Key Features:

1. User & Role Management
- A distributor can have multiple users (logins)
- Roles include:
  - Admin
  - Biller
  - Accountant
- Role-based access control should be implemented

2. Distributor (Company) Profile
Each distributor has:
- Company Name
- Logo
- Address
- Phone Number
- GST Number
- Drug License Number
- Bank Details

This information is dynamically used in invoices and reports.

3. Customer Management
- Customers are businesses (pharmacies, clinics)
- Each customer has:
  - License Number (Primary Identifier)
  - Name
  - Address
  - GST Number
  - Contact Details

4. Product Management
- Products include:
  - Product Name
  - Batch Number
  - Expiry Date
  - MRP
  - Rate
  - GST %
- Inventory tracking is required

5. Billing System (Core Feature)
- Generate Tax Invoices
- Each invoice includes:
  - Distributor details (dynamic based on logged-in user)
  - Customer details
  - Multiple products
  - GST calculations (SGST + CGST)
  - Discounts
- Invoice numbering should be unique per distributor
- Support Credit/Cash invoices

6. Invoice Output
- Printable A4 format
- Export to PDF
- Professional layout (similar to real pharma invoices)
- Include:
  - Logo
  - Tax breakdown
  - Amount in words
  - Authorized signature

7. Database Design (High-Level)
Entities:
- Users (Distributor + staff logins)
- Roles
- Customers (with license_no as key)
- Products
- Invoices
- Invoice_Items

8. Multi-Tenant Logic (IMPORTANT)
- Each distributor must only access their own:
  - Customers
  - Products
  - Invoices
- Data isolation is mandatory

9. Offline + Online Capability
- System should work offline (local DB like SQLite/MySQL)
- Future-ready for cloud sync

10. Technology Stack
- Python-based (as per requirements.txt)
- Desktop UI (Tkinter / PyQt / Custom UI framework)
- Database: SQLite/MySQL
- PDF generation supported

11. Design Expectations
- Clean, minimal, professional UI
- Fast billing workflow
- Error-free calculations
- Scalable architecture

Goal:
Build a reliable, scalable, and user-friendly pharmaceutical billing and distribution system that can be used by multiple distributors independently with their own branding and data.

Important Notes:
- Do NOT treat customers as walk-in buyers
- License number is the key identifier for customers
- One distributor = one data boundary
- Multiple users per distributor must be supported
